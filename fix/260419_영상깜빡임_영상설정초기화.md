# 미해결 버그 인수인계 — 260419

## 1. 영상 깜빡임 (미해결)

### 증상
- 테스트 영상 파일 재생 중 영상 위젯 내부만 깜빡임 (나머지 UI는 정상)
- 최초 실행 직후부터 발생

### 다음 세션에서 할 것
- 사용자가 녹화 파일을 첨부해서 설명할 예정 → 영상 보고 재진단
- 깜빡임이 주기적인지(파일 루프 포인트마다), 불규칙한지 확인 필요

### 현재 코드 상태 (이번 세션에서 변경된 내용)
`ui/video_widget.py` — 이번 세션에서 두 차례 큰 변경을 했지만 여전히 미해결

**현재 구조 (최종 상태)**
- `QLabel.setPixmap()` 방식 → `paintEvent` 직접 그리기로 전환 완료
- `WA_OpaquePaintEvent=True` 설정 (배경 지우기 스킵)
- `update_frame()` 호출 시 `_rebuild_pixmap()` → `self.update()` 예약
- `resizeEvent`에서 `self.update()`만 호출 (pixmap 재빌드 없음)
- `set_blink_state()` — 알람 없으면 `update()` 호출 안 함

**시도했으나 효과 없었던 것들**
- `QImage.tobytes()` GC 문제 수정 (rgb_bytes 변수로 보관)
- `_cached_pixmap` 도입 (프레임 변경 시에만 BGR→RGB 변환)
- `SharedFramePoller`에서 `read_frame() is None` 시 `clear_signal()` 제거

### 의심 지점 (다음 세션에서 확인)
- `_rebuild_pixmap()` 이 `update_frame()` 호출마다 실행되는데, 33ms마다 BGR→RGB 변환 + `QPixmap.fromImage()` 가 메인 스레드에서 실행됨. 이게 UI 이벤트 루프를 막아 렌더 타이밍이 불규칙해질 수 있음
- `video_capture.py` 파일 재생 시 `time.sleep(1/30)`이 있지만 `cap.read()` 자체가 파일에서는 블로킹 없이 즉시 반환하므로 실제 fps 제어가 안 될 수 있음 → seq_no가 33ms 사이에 여러번 바뀌어도 UI는 33ms에 한 번만 처리하므로 이건 문제 아님
- `paintEvent`에서 `drawPixmap(QRect(...), self._display_pixmap)` — 스케일링을 Qt에 맡기는데, 이 변환이 매 paintEvent마다 일어남. `SmoothTransformation` vs `FastTransformation` 차이 확인 필요
- 근본적으로 BGR→RGB + QPixmap 생성을 별도 스레드로 옮기는 게 필요할 수 있음

## 2. 영상 설정 초기화 버튼 후 검은 화면 (미해결 가능성)

### 증상
- 설정창 > 영상탭 > "파일 입력(테스트용)" 의 **초기화** 버튼을 누르면
- 테스트 영상이 사라지고 검은 화면이 보여야 하는데 안 됨

### 이번 세션에서 한 수정
`ui/main_window.py` `_on_config_saved()` 에 아래 추가:
```python
new_video_file = new_cfg.get("video_file", "")
if old_video_file and not new_video_file:
    self._video_widget.clear_signal()
```
→ 설정이 저장될 때 video_file이 비워졌으면 VideoWidget을 검은 화면으로 전환

### 미확인 상태
- 이 수정이 실제로 동작하는지 아직 테스트 못 함
- Detection이 재연결 중에 새 프레임을 계속 SharedMemory에 쓰면 `clear_signal()` 직후 다시 영상이 표시될 수 있음
- 필요 시 Detection 재연결 완료까지 UI 측에서 프레임 수신을 일시 차단하는 로직 추가 고려
